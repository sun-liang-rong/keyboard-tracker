# 项目需求文档 — 键盘活跃度统计工具

> **📖 开发文档**: [DEVELOPMENT.md](./DEVELOPMENT.md) - 包含详细的技术规范、API接口、数据模型、开发规范等

## 1. 项目概述

| 项目属性 | 内容 |
|---------|------|
| **项目名称** | KeyboardTracker（键盘活跃度统计工具）|
| **项目类型** | 桌面端应用程序 |
| **支持平台** | Windows / macOS |
| **目标用户** | 程序员、打工人、学生等长时间使用电脑的人群 |
| **产品定位** | 记录用户键盘使用情况，统计每日/每周/每月敲击次数和时间段分布，用数据量化专注度和工作效率 |

## 4. 数据模型

详细的数据模型定义请参考 [DEVELOPMENT.md](./DEVELOPMENT.md#5-数据模型)。

### 4.1 核心数据结构

```typescript
// 按键分类统计
interface KeyCategoryCount {
  letter: number      // 字母键
  number: number      // 数字键
  function: number    // 功能键
  control: number     // 控制键
  symbol: number      // 符号键
  modifier: number    // 修饰键
  other: number
}

// 组合键统计
interface ComboCounts {
  COPY: number
  PASTE: number
  CUT: number
  // ... 更多组合键
}

// 每日统计
interface DailyStat {
  date: string
  totalCount: number
  hourlyDistribution: number[]  // 24小时分布
  activeHours: number
  focusSessions: number
  categoryCount: KeyCategoryCount
  topKeys: TopKeyItem[]
  comboCounts: ComboCounts
}
```

## 7. 技术难点与解决方案

详细的技术实现请参考 [DEVELOPMENT.md](./DEVELOPMENT.md#2-技术架构)。

### 7.1 键盘监听方案

采用**平台原生 CLI 工具**方案：

| 平台 | 方案 | 说明 |
|-----|------|------|
| **macOS** | Quartz / CGEvent | 编译为 CLI 工具，Node.js 通过 child_process 调用 |
| **Windows** | AutoHotkey | AHK 脚本编译为 exe，Node.js 调用 |

### 7.2 方案优势

| 对比项 | iohook | 原生 CLI 方案 |
|-------|--------|--------------|
| 依赖管理 | 复杂 | 无依赖，预编译二进制 |
| 跨平台 | 需单独配置 | 各平台独立实现 |
| 权限申请 | 运行时动态申请 | 安装时一次性授权 |
| 性能 | 中等 | 高（原生代码）|

### 7.3 项目结构

详细目录结构请参考 [DEVELOPMENT.md](./DEVELOPMENT.md#4-项目结构)。

```
KeyboardTracker/
├── src/
│   ├── main/              # Electron 主进程
│   ├── renderer/          # Vue 3 前端
│   └── bin/               # Native 二进制文件
├── native/                # Native 工具源码
├── public/                # 静态资源
└── package.json
```

### 7.4 其他技术难点

| 难点 | 解决方案 |
|-----|---------|
| 跨平台兼容性 | Electron 封装，平台差异代码分离处理 |
| 数据读写性能 | lowdb 使用 lodash 链式操作，大数据量时定期归档 |
| 后台运行保活 | 注册系统服务 / Tray 托盘保持运行 |
| 安装包权限 | 安装时申请辅助功能/管理员权限（避免运行时弹窗）|

详细解决方案请参考 [DEVELOPMENT.md](./DEVELOPMENT.md)。

## 8. 隐私与安全

- **本地存储**：所有数据仅存储在用户本地，不上传云端
- **内容保护**：只记录按键次数，绝不记录具体按键内容
- **数据清理**：支持自动清理过期数据（可配置保留天数）
- **权限最小化**：仅申请键盘监听必要权限
